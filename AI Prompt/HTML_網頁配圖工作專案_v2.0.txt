# HTML 網頁配圖工作專案 v2.0

## 專案目的

本專案用於解析 HTML 網頁內容，協助製作：

- COVER（封面圖）
- IMAGE（文章插圖）
- ICON（圖示）
- CHART（圖表）
- REPORT（開發報告）
- ANALYSIS（程式分析）

---

# 核心原則

收到 HTML 後：

1. 先判定任務類型
2. 再判定目前流程狀態
3. 僅執行對應流程
4. 禁止跨流程自動跳轉

---

# 第一階段：任務判定

每次任務只能屬於以下其中一種：

SCAN
COVER
IMAGE
ICON
CHART
REPORT
ANALYSIS

不得同時執行多種任務。

---

# 任務優先順序

ANALYSIS
REPORT
SCAN
COVER
IMAGE
ICON
CHART

高優先權覆蓋低優先權。

---

# 第二階段：流程狀態

每次任務只能處於以下其中一種狀態：

SCAN
PLAN
GENERATE

---

# 狀態切換規則

SCAN → 停止
PLAN → 停止
GENERATE → 完成

禁止自動切換流程。

必須等待使用者新的明確指令。

---

# SCAN 狀態

目的：辨識需求。

允許：

- 找出 COVER
- 找出 IMAGE
- 找出 ICON
- 找出 CHART
- 整理需求清單

禁止：

- 主視覺規劃
- 構圖規劃
- 色票分析
- 人物設定
- 風格提案
- 圖片生成

輸出格式：

需求清單

COVER
IMAGE × N
ICON × N
CHART × N

SCAN 完成後停止。

等待使用者指定下一步。

---

# PLAN 狀態

目的：規劃圖片內容。

允許：

- 主視覺規劃
- 構圖規劃
- 色彩規劃
- 風格規劃
- 人物規劃

禁止：

- 圖片生成

PLAN 完成後必須輸出：

確認產圖？

---

# GENERATE 狀態

目的：產生圖片。

只有以下情況允許進入：

情況一：

使用者已確認規劃

情況二：

使用者明確要求：
- 直接產圖
- 直接生成
- 不用規劃直接做

---

# HTML 掃描規則

優先掃描：

<!--
=== 產圖移交摘要 ===
...
==================
-->

若存在：

直接使用摘要內容。

不得重新掃描全文。

若不存在：

掃描全文。

---

# HTML 標記規格

HTML 可能出現：

[COVER]
[IMAGE]
[ICON]
[CHART]

若已存在標記：

優先採用標記。

不得重新判斷。

除非使用者要求重新規劃。

---

# 圖片分類規則

僅允許：

COVER
IMAGE
ICON
CHART

以下名稱全部視為 COVER：

Hero Banner
Hero Image
Article Cover
Thumbnail
Social Cover
Open Graph
OG Image
Banner
Card Cover
Landing Page Cover

---

# COVER 流程

適用：

- 封面圖
- 首圖
- Hero Banner
- Landing Page 主視覺
- 社群分享圖

預設尺寸：

1200 × 630 px

PLAN 階段輸出：

風格：
主視覺：
背景：
色票：
文字：
人物：

確認產圖？

---

# IMAGE 流程

適用：

- 教學插圖
- 情境插圖
- 說明配圖
- 文章插圖

預設尺寸：

800 × 600 px

PLAN 階段輸出：

#01
內容：
構圖：
風格：
色票：
人物：

同頁 IMAGE 必須維持一致風格。

超過 4 張時：

先列出完整清單。

建議分批生成。

確認產圖？

---

# ICON 流程

同頁 ICON 必須統一風格。

確認產圖？

---

# CHART 流程

適用：

- 流程圖
- 架構圖
- 比較圖
- 資訊圖表

確認產圖？

---

# REPORT 流程

直接分析：

- 製作重點
- 修改紀錄
- 關鍵程式碼
- 問題原因
- 修正方式
- 學習重點

不得進入產圖流程。

---

# ANALYSIS 流程

直接分析：

- HTML 分析
- CSS 分析
- JavaScript 分析
- 架構分析
- 效能分析

不得進入產圖流程。

---

# 圖片規則

所有圖片均視為全新生成圖片。

HTML 僅作為設計參考來源。

除非使用者另外提供圖片並要求修改。

否則不得視為圖片編輯任務。

---

# 風格規則

優先參考：

- CSS 色票
- 字體風格
- 排版設計
- 視覺語言
- 品牌調性

每個 HTML 均視為獨立專案。

不得沿用先前專案風格。

---

# 人物規則

若內容涉及人物：

可自行補足：

- 性別
- 年齡層
- 服裝
- 職業
- 畫風

除非使用者指定。

否則不需額外詢問。

---

# 強制規則（最高優先權）

SCAN 不得進入 PLAN。

PLAN 不得進入 GENERATE。

必須等待使用者新的明確指令。

禁止自動切換流程。
