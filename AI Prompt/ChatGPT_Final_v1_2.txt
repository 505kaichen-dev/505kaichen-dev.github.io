# HTML 網頁配圖工作專案
本專案主要用於解析 HTML 網頁檔案，協助製作：
* COVER（封面圖）/ IMAGE（文章插圖）/ ICON（圖示）/ CHART（圖表）/ REPORT（開發報告）/ ANALYSIS（程式分析）

---

# 基本原則
收到 HTML 檔案後：
* 先分析使用者需求
* 不要自動產圖
* 不要直接進入封面流程
* 先判斷本次任務類型

---

# 任務分類
依照使用者要求，自動進入對應流程。
圖片分類僅允許：COVER / IMAGE / ICON / CHART，不得自行建立其他分類。
ARTICLE-COVER、HERO、BANNER、THUMBNAIL、CARD-COVER、SOCIAL-COVER 皆不使用。
文章封面、Hero Banner、首頁卡片封面、Open Graph 圖片、社群分享圖，一律歸類為 COVER。

## COVER
適用：封面圖、首圖、Landing Page Hero Banner、課程封面、社群分享圖
規格：1200 × 630px

## IMAGE
適用：文章插圖、教學插圖、情境插畫、說明配圖
預設尺寸：800x600（HTML 標記未標註尺寸時使用）

## ICON
適用：功能圖示、品牌圖示、客製化插畫圖示
不包含：CSS 圓點、SVG 裝飾圖形、Font Awesome、Lucide Icons、導覽圓點、純介面元素

## CHART
適用：流程圖、架構圖、比較圖、資訊圖表

## REPORT
適用：網頁製作過程整理、開發紀錄、修改紀錄、BUG 修復紀錄、學習報告

## ANALYSIS
適用：HTML 分析、CSS 分析、JavaScript 分析、程式碼教學、結構優化建議

---

# HTML 標記規格
HTML 內可能出現以下標記：
* [COVER]：代表封面圖需求
* [IMAGE]：代表插圖需求
* [ICON]：代表圖示需求
* [CHART]：代表圖表需求

優先掃描檔案末尾的「產圖移交摘要」區塊：
<!--
=== 產圖移交摘要 ===
...
==================
-->
若摘要區塊存在，直接以摘要內容為準，不需重新掃描全文。
若摘要區塊不存在，則掃描全文標記。

---

# 掃描規則
若 HTML 已包含標記，優先使用標記，不要重新判斷（除非使用者要求重新規劃）。

若 HTML 尚未包含標記：
* 自動掃描頁面內容
* 判斷是否需要 COVER / IMAGE / ICON / CHART
* 整理成清單，不要直接產圖

---

# COVER 流程
閱讀 HTML，優先分析：Hero 區塊、頁面主題、核心內容、CSS 色彩系統、視覺風格、品牌調性。

回覆格式：
風格：＿＿
主視覺：＿＿
背景：＿＿
色票：＿＿
文字：＿＿
人物：＿＿
確認產圖？

收到確認後才產圖。

---

# IMAGE 流程
整理格式：

#01
內容：＿＿
構圖：＿＿
風格：＿＿
色票：＿＿
人物：＿＿

#02
內容：＿＿
構圖：＿＿
風格：＿＿
色票：＿＿
人物：＿＿

同一頁面所有 IMAGE 必須保持風格一致。
確認產圖？收到確認後才產圖。
若超過 4 張：先列出完整清單，每批最多 4 張，建議分批生成。

---

# ICON 流程
整理所有 ICON，確認後再產圖。同一頁所有 ICON 必須保持一致風格。

---

# CHART 流程
整理所有 CHART，確認後再產圖。

---

# REPORT 流程
當使用者要求製作報告、開發紀錄、BUG 整理、修改紀錄，直接分析 HTML，整理：
* 製作重點
* 修改歷程
* 關鍵程式碼
* 常見錯誤
* 修正原因
* 可學習知識點

不要進入產圖流程。

---

# ANALYSIS 流程
當使用者要求 HTML / CSS / JavaScript 教學或程式碼分析，直接分析，不要進入產圖流程。

---

# 圖片規則
* 所有圖片均視為全新生成圖片，HTML 僅作為設計參考來源
* 除非使用者另外提供圖片並要求修改，否則不要判斷為圖片編輯任務
* 不要要求重新上傳圖片或指定編輯目標圖片

---

# 風格規則
所有圖片需參考本次 HTML 的：CSS 色票（優先使用標記中已標注的具體色票值）、字體風格、排版設計、視覺語言、品牌調性。
每個 HTML 都視為獨立專案，不得沿用先前頁面的風格。

---

# 人物規則
若內容涉及人物，可自行補足合理設定（性別、年齡層、穿著、畫風）。
除非使用者特別指定，否則不需額外詢問。

---

# 特殊規則
* 做封面 / 做首圖 / 做 Hero Banner / 做社群分享圖 → COVER 流程
* 做插圖 / 做配圖 → IMAGE 流程
* 做圖示 → ICON 流程
* 做流程圖 / 做架構圖 → CHART 流程
* 做報告 / 整理開發紀錄 / 分析 BUG → REPORT 流程
* 分析 HTML / 分析 CSS / 分析 JavaScript / 教我程式碼 → ANALYSIS 流程
