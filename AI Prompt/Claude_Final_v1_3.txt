這是我的個人網站 505kaichen-dev.github.io 的開發專案。

【首頁設計系統】（只適用於 index.html 和 gallery/index.html）
* 純 HTML5，CSS 和 JS 寫在同一個檔案
* 風格：科技感、淺色質感
  --bg: #F8F9FC / --blue: #2563EB / --ink: #0D0F14
  字型：Space Grotesk（標題）/ Inter（內文）
* 最大寬度 1280px 置中，響應式斷點 900px / 600px

【子頁面】
每個子頁面風格自由，沒有限制，依內容主題決定設計方向。

【子頁面最低結構要求】
每個子頁面至少需包含：
* header（含返回首頁連結，位置左上角，連結到 ../）
* main content 區塊
* footer
不得省略上述三個結構，其餘版面自由發揮。

【資料夾結構】
根目錄/
├── index.html（首頁）
├── gallery/（AI圖像創作集）
├── opensource-story/
├── claude-web-guide/
├── dradvice01/（密碼保護：health2025）
├── LocalAI_Image/
└── ai-guide/

【封面圖規格】
1200×630px，主要內容置中
放在各資料夾的 image/ 子資料夾
Gallery 引用路徑用 ../資料夾/image/檔名

【個人風格定位】
不正經的極客，理性但幽默，好奇心驅動
首頁文案基調：「不是部落格。不是作品集。就是我覺得很酷所以做了。」

【新增子頁面流程】
1. 建立資料夾（命名規則：客戶用 品牌+編號，自己的內容用語意化英文）
2. 製作封面圖放進 image/
3. 更新首頁 index.html 新增卡片
4. 客戶作品加 data-password 屬性

【密碼保護實作規範】
客戶作品頁面的密碼驗證邏輯，必須沿用以下固定模板，不得自行改寫：
* 在 <body> 最上層插入遮罩層 #password-overlay
* 驗證邏輯寫在同一個 HTML 檔案的 <script> 區塊
* 密碼錯誤顯示固定提示文字：「密碼錯誤，請重試」
* 驗證成功後移除遮罩層，不做頁面跳轉
* data-password 屬性值即為明文密碼，JS 直接比對

【子頁面規範】
每個子頁面都要有一個回首頁的按鈕或連結，
位置在左上角或導覽列，連結到 ../（上一層根目錄）。

【開發流程規範】
依修改規模決定是否走確認流程：

小幅修改（免確認，直接執行）：
* 修改文案、標題、描述文字
* 調整色票數值
* 修正錯字或連結

中大幅修改（必須走完整確認流程）：
* 新增頁面或資料夾
* 新增頁面區塊或功能模組
* 調整整體版面結構
* 新增互動元素（動畫、篩選、彈窗等）

完整確認流程如下：
1. 草圖確認階段
   * 用文字草圖呈現版面架構（區塊位置、內容形式、互動元素）
   * 列出待確認的設計決策點
   * 等待 Kai 明確回覆確認或修改
2. 內容確認階段
   * 確認所有核心內容清單（章節、主題、文案方向）
   * 確認資料夾命名、檔案結構
3. 開工指令
   * 只有在 Kai 明確說「可以開始做」、「開工」或類似指令後才開始產出程式碼或檔案
   * 中途有任何架構疑問，先問再做，不自行假設

【禁止行為】
* 不在未確認架構前產出完整程式碼（適用中大幅修改）
* 不在未收到開工指令前進入實作階段（適用中大幅修改）

【圖片標註規範】
HTML 中需要圖片素材的位置，請主動加入圖片標記，方便後續交由 ChatGPT 掃描與產圖。
圖片描述語言統一使用繁體中文。

---

## 封面圖
用途：子頁文章主封面、首頁文章卡片封面、Open Graph 圖片、社群分享圖、Landing Page Hero Cover
格式：<!-- [COVER] 1200x630 圖片描述 -->
範例：<!-- [COVER] 1200x630 科技感淺色風格 AI 學習指南封面，發光瀏覽器視窗、程式碼卡片、藍白配色（#2563EB / #F8F9FC）、玻璃擬態介面 -->

## 插圖
用途：文章配圖、教學示意圖、情境插畫、說明配圖
格式：<!-- [IMAGE] 800x600 圖片描述 -->
尺寸未指定時預設使用 800x600。
範例：<!-- [IMAGE] 800x600 年輕開發者在多螢幕工作環境整理 AI 工具流程，科技感淺色介面風格，藍白配色（#2563EB / #F8F9FC） -->

## 圖示
用途：功能特色圖示、品牌特色圖示、客製化插畫圖示
格式：<!-- [ICON] 圖片描述 -->
範例：<!-- [ICON] 程式碼括號與星光元素，代表創意開發，藍白配色，線條簡潔扁平風格 -->

## 圖表
用途：流程圖、架構圖、比較圖、資訊圖表
格式：<!-- [CHART] 圖片描述 -->
範例：<!-- [CHART] HTML → ChatGPT → 圖片生成 → 網站上線流程圖，藍白配色，簡潔箭頭連接 -->

【圖片標記規則】
* 圖片標記必須使用 HTML 註解
* 僅允許使用：[COVER] / [IMAGE] / [ICON] / [CHART]
* 不得建立其他圖片分類
* COVER 必須使用 1200x630
* IMAGE 需標註建議尺寸（800x600 / 600x400 / 500x500），未指定預設 800x600
* ICON 與 CHART 可不標註尺寸

【圖片描述規則】
圖片描述應盡量具體，必須包含：
* 畫面主體
* 情境
* 視覺風格
* 色彩方向（盡量寫出具體色票，例如 #2563EB 藍、#F8F9FC 淺灰）
* 人物設定（如有）

避免：<!-- [IMAGE] AI工具 -->
建議：<!-- [IMAGE] 800x600 開發者在多螢幕工作環境中整理 AI 工具流程，藍白科技感介面（#2563EB / #F8F9FC），玻璃擬態風格 -->

【封面圖規則】
* 每個子頁面預設至少產生一個 COVER 標記
* COVER 必須依據該頁面主題設計，不要根據首頁卡片文案猜測內容
* 首頁僅引用各子頁 image 資料夾中的封面圖，不要額外建立首頁專用封面圖

【風格規則】
若頁面已有 CSS 色票、字體風格、品牌調性，
圖片描述必須將色票數值具體寫入標記中，不得只寫「風格跟隨本頁 CSS 設計語言」。

【產圖移交摘要】
每次完成 HTML 後：
1. 在 HTML 檔案最末尾加入以下區塊：

<!--
=== 產圖移交摘要 ===
本次頁面圖片需求如下，供 ChatGPT 掃描使用：

[COVER] ...
[IMAGE] ...
[ICON] ...
[CHART] ...

==================
-->

2. 額外產出一份獨立的 [頁面名稱]_圖片移交.txt，內容只包含產圖移交摘要區塊，供直接移交給 ChatGPT 使用，不需要傳完整 HTML 檔案。

若該頁面無圖片需求，則兩者皆省略。

【禁止事項】
不要使用：ARTICLE-COVER / HERO / BANNER / THUMBNAIL / CARD-COVER / SOCIAL-COVER
不要使用：<!-- [插圖] 尺寸：＿＿px 內容：＿＿ -->
