怎麼跟 GPT 說
才能拿到對的程式碼？

這是最關鍵的一步。你要跟 AI 要兩樣東西：一段 GAS 程式碼，以及一個 HTML 表單頁面。

505kaichen-dev.github.io — 原始網頁 AI 做網頁進階課
Ch.4

分兩步驟跟 GPT 要程式碼

1

第一步：要 GAS 程式碼

告訴 GPT 你的 Sheets ID、工作表名稱、表單欄位。GPT 會給你一段要貼到 script.google.com 的程式碼。

2

部署 GAS → 拿到 Web App URL

把程式碼貼進去、部署成 Web App，Google 會給你一串很長的網址（Web App URL）。

3

第二步：要 HTML 表單，並填入 URL

把 URL 告訴 GPT，請它做 HTML 表單頁面並把 URL 填進去。這樣表單送出時才知道要傳到哪裡。

💡
為什麼要分兩步驟？

GAS 的 Web App URL 要先部署才會有。你沒辦法在問 HTML 的時候就把 URL 填進去，所以順序是：先 GAS → 部署 → 拿 URL → 再叫 GPT 做 HTML。

Ch.5

GAS 部署五步 + HTML 完成四步

⚙️ 步驟一：部署 GAS
  1. 打開 script.google.com登入 Google，點「新增專案」
  2. 貼上 AI 給的程式碼全選貼進去，覆蓋空白內容
  3. 點「部署」→「新增部署作業」類型選「網頁應用程式」
  4. 存取權限設定「所有人」這樣訪客送出時才能成功傳資料
  5. 點「部署」，複製 Web App URL這串網址很長，完整複製備用
📄 步驟二：完成 HTML
  1. 回到 GPT 對話告訴 GPT：「GAS 已部署，Web App URL 是：[貼上 URL]，請把 URL 填進 HTML 裡」
  2. GPT 會給你最終版 HTML把完整的 HTML 程式碼複製下來
  3. 存成 .html 檔案新建文字檔，貼上程式碼，副檔名改成 .html
  4. 用瀏覽器打開雙擊那個 .html 檔案，就能看到你的表單頁面了

本地測試：
不用上架就能驗證

網頁不需要上傳到 GitHub 或伺服器才能測試。在你的電腦上直接開 HTML 就能正常運作。

505kaichen-dev.github.io — 原始網頁 AI 做網頁進階課
Ch.6

四步驗收，確認資料有進 Sheets

  • 1
    在電腦上雙擊打開 .html 檔案 瀏覽器網址列會顯示 file:///Users/...，這是正常的
  • 2
    填寫一筆測試資料送出 用假資料測試，例如姓名填「測試001」，這樣等一下在 Sheets 看到就知道是測試資料
  • 3
    確認頁面出現成功訊息 網頁應該會顯示「感謝您的報名」或類似的提示
  • 4
    去 Google Sheets 確認資料有進來 打開試算表，應該可以看到剛才填的那筆資料出現在第二行
🎉
看到資料進 Sheets 了？恭喜你成功了！

接下來就可以把 HTML 檔案交給負責上架的人，或是直接丟到 GitHub Pages，功能完全一樣。

Ch.7

三件事不能忘，避免個資風險

🗑️

清除 Google Sheets 裡的測試資料

打開試算表，選取所有測試用的資料行，右鍵 → 刪除列。保留標題行就好。

🔌

關閉 GAS 的 Web App 部署

script.google.com → 找到你的專案 → 部署 → 管理部署作業 → 封存或刪除。關掉後那個 URL 就失效了。

📋

活動真實資料的處理

和公司確認資料要保留多久、由誰管理，依台灣個資法規定處理，不需要的資料要刪除。

⚖️
記得在網頁上加一行隱私權說明

請 AI 在表單下方加小字：「本活動蒐集之個人資料僅供本次體驗活動使用，不對外提供第三方。」

CORS 問題示範對話

GPT 通常會回
好的，我來幫你檢查幾個常見原因：
1. 請確認你的 Sheets ID 有沒有貼對……
2. 部署時有沒有點到「更新」……
3. 可以打開開發者工具（F12）→ Console，告訴我紅色錯誤訊息寫什麼……

Ch.8

問 GPT Debug 的三個關鍵資訊

有這三樣，GPT 幾乎都能幫你找出問題：

①

你做了什麼

照哪些步驟操作？GAS 的部署設定是什麼？

②

你看到什麼

網頁顯示什麼訊息？是「送出失敗」、「感謝」還是頁面沒有反應？

③

Console 的錯誤訊息

按 F12 → Console 標籤，截圖或複製紅色文字。這是最關鍵的線索。

🔍
Debug 心法

不要只說「沒有用」或「出錯了」。給 GPT 上面三樣，它才能幫你精準定位問題所在。
