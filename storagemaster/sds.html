SDS 核心思想#

Software-Defined Storage（SDS）的核心是把儲存的管理邏輯、資料服務（複製、快照、重刪、壓縮）從特定硬體中解放出來，跑在通用 x86 伺服器或雲端環境上。採購時不再被特定廠商的硬體綁定，底層硬體可以替換，上層服務不受影響。

SDS vs 傳統儲存的根本差異

面向傳統儲存（Appliance）軟體定義儲存（SDS）
智慧在哪裡控制器韌體，與硬體不可分離軟體層，可跑在任何相容硬體
硬體依賴高（換硬體 = 換廠商）低（底層硬體可替換）
擴展方式向廠商購買特定規格硬體加入通用 x86 節點即可擴展
初始成本較高（專用硬體 + 授權）較低（通用硬體 + 軟體授權）
效能可預測性高（廠商驗證的固定組合）依硬體組合而異，需自行調優
適合場景使命關鍵、高可用性需求、不願冒風險的核心系統雲原生、容器化、AI 訓練、成本敏感、私有雲建設

三種 SDS 路線

🔧 傳統廠商 SDS
IBM、Dell、NetApp 等傳統儲存廠商推出的 SDS 版本，通常是其 Appliance 產品的軟體化，保留相同管理介面和功能集，有商業支援。
代表：IBM Storage Scale / Ceph、Dell PowerFlex、NetApp ONTAP Select
🌱 開源生態
Ceph、MinIO、Longhorn、Rook 等開源專案，免費使用，社群活躍，但需要自行維護，或採購商業支援版本（如 Red Hat Ceph Storage）。
代表：Ceph / Rook / MinIO / Longhorn
🚀 新架構平台
VAST Data、HPE Alletra MP 等採用 DASE（解耦架構）的新世代平台，打破傳統控制器和磁碟綁定的設計，專為 AI 和雲原生場景設計。
代表：VAST Data / HPE Alletra MP B10000 / NetApp AFX
重要認知：SDS 不等於「便宜」
SDS 降低的是硬體的廠商鎖定，但軟體授權、維運人力、調優成本不見得比傳統 Appliance 低。尤其開源 SDS 的總擁有成本（TCO），需要把人力成本算進去才能公平比較。

IBM SDS 與 FlashSystem 的搭配

IBM SDS 產品通常不是獨立部署，而是與 FlashSystem 搭配：FlashSystem 提供高效能 Block 儲存，Storage Scale 在上面建立高吞吐量的 File 層，讓 AI 訓練框架（PyTorch、TensorFlow）能直接高速存取訓練資料集。這個組合是 IBM 在 AI 基礎設施市場的主要方案。

VMware vSAN#

vSAN 是目前企業市場部署最廣泛的 SDS / HCI 儲存解決方案，把 ESXi 叢集中每台主機的本機 SSD 聚合成共享儲存池，完全整合在 vSphere 管理介面中。

vSAN 的核心設計

  • 本機儲存聚合：每台 ESXi 主機的 NVMe / SSD 通過 vSAN 軟體聚合成叢集共享儲存，不需要獨立的 SAN 設備
  • 物件儲存引擎：vSAN 在底層把 VMDK 切成物件（Component），分散在多台主機上保護，不是傳統 RAID 概念
  • Storage Policy Based Management（SPBM）：以策略（Policy）定義 VM 的儲存要求（副本數、效能層），vSAN 自動執行
  • Stretched Cluster：跨兩個站點的 vSAN 叢集，提供雙活能力，需要 Witness 節點仲裁

vSAN 版本演進

版本重要特性
vSAN 傳統架構Hybrid（SSD Cache + HDD）或 All-Flash，以 RAID-1 / RAID-5 / RAID-6 保護物件
vSAN ESA（Express Storage Architecture）2022 推出的全新架構，拋棄舊有 Hybrid 設計，All-NVMe 專屬；單層儲存池（不再區分 Cache / Capacity）；效能大幅提升；需要 vSAN ESA 認證硬體

vSAN 的適用場景與限制

適合不適合
  • 已有大量 VMware 環境，想整合儲存
  • VDI（虛擬桌面）
  • 非核心應用 / 測試環境
  • 邊緣節點 / 遠端辦公室
  • 需要獨立擴展計算和儲存的場景
  • IBM i / AIX 等非 x86 工作負載
  • 超高 IOPS 核心資料庫（通常建議傳統 SAN）
  • 預算有限但節點數少（授權成本高）

Dell PowerFlex#

PowerFlex（前身 ScaleIO）是 Dell 的純軟體定義 Block 儲存，可以跑在通用 x86 伺服器上，也有 Dell Appliance 版本。2025 年推出的 PowerFlex Ultra 大幅提升效率與可用性。

PowerFlex 架構特點

  • 純 Block 儲存：只提供 Block（iSCSI / NVMe-oF），不提供 File 或 Object，定位清晰
  • 軟體部署彈性：可跑在任何 x86 伺服器，也可買 Dell Appliance 版本（硬體驗證，部署更快）
  • PowerFlex Ultra（2025）：10個9資料可用性；80% 儲存效率；Dell Private Cloud 原生支援，與 PowerStore / PowerMax 並列
  • OpenStack / Kubernetes 整合：有 Cinder Driver 和 CSI Driver，是大型 OpenStack 私有雲的常見儲存選項

對標 vSAN 和 Ceph

比較項目Dell PowerFlexVMware vSANCeph（開源）

Hypervisor 依賴無（Hypervisor 無關）強依賴 VMware ESXi無

⚠️ 開源 SDS 的維運成本

HCI 陣營（Nutanix / VxRail）#

HCI（超融合）把計算和儲存融合在同一節點，儲存層由軟體（SDS）提供。雖然對終端用戶看起來是買一台「超融合設備」，底層其實是跑在 x86 上的 SDS。

產品廠商Hypervisor儲存軟體特色
Nutanix AOS Nutanix AHV（自研）/ ESXi / Hyper-V Nutanix AOS（分散式儲存引擎） HCI 發明者；管理介面最友善；AHV 是免費 Hypervisor；Nutanix Files / Objects 提供 NAS / S3 延伸
Dell VxRail Dell ESXi（VMware 專屬） VMware vSAN Dell 硬體 + VMware vSAN，整合認證；適合已有 VMware 授權的客戶
HPE SimpliVity HPE ESXi HPE 自研（含硬體壓縮加速晶片） 內建硬體壓縮加速，備份和複製效能強

HCI vs 傳統 SAN 的思維轉換

採用 HCI 最大的思維轉換是：儲存不再是獨立的設備，而是叢集的一部分。擴展計算就是擴展儲存，擴展儲存就是擴展計算。這在計算和儲存成長比例相近的環境很有效率，但在兩者成長比例差異大時（例如儲存快速成長但計算不需要增加）就會有浪費。

DASE 新架構（VAST / NetApp AFX / HPE Alletra MP）#

DASE（Disaggregated Shared Everything）是近年最受關注的新儲存架構，解決了 HCI「計算和儲存必須同步擴展」的問題，同時保留 SDS 的靈活性，是 AI 時代儲存架構的重要方向。

DASE vs 傳統架構的差異

產品廠商架構最強場景
VAST Data Platform VAST Data（新創） 完全解耦 DASE，Global Namespace AI 訓練 / 推理；Block + File + Object + DB 四合一；GPUDirect 最佳化
HPE Alletra MP B10000 HPE DASE，控制平面與資料平面分離 AI 工作負載；大規模虛擬化；NVMe-oF 後端
NetApp AFX NetApp 解耦架構，NVMe-oF + RoCE 後端 高效能 NAS / S3；AI / ML 資料管線；HPC 工作負載

選型判斷#

SDS 的選型沒有標準答案，取決於工作負載特性、既有環境、人力能力和預算。以下是常見場景的建議方向。

🏦
企業核心應用（資料庫、ERP）
→ 傳統 SAN Appliance 優先
IBM FlashSystem / Dell PowerStore / NetApp AFF。SDS 在使命關鍵場景的可靠性和效能保證仍不如成熟的 Appliance，除非已有 SDS 專業能力。
☁️
私有雲 / OpenStack
→ Ceph 或 PowerFlex
Ceph 是 OpenStack 的事實標準儲存後端；若在 Dell 環境且想要商業支援，PowerFlex 是成熟選項。
🐳
Kubernetes 容器儲存
→ Rook+Ceph / Longhorn / 傳統 SAN CSI
小型叢集用 Longhorn（輕量）；中大型用 Rook+Ceph；若已有企業 SAN，各品牌 CSI Driver 是最簡單的整合方式。
🤖
AI 訓練資料儲存
→ VAST Data / IBM Storage Scale / MinIO
需要高吞吐量 + GPUDirect 支援。VAST Data 是新創首選；IBM Storage Scale 是 HPC 成熟方案；MinIO 適合 S3 為主的 AI pipeline。
💻
VMware 環境整合
→ vSAN 或 VxRail
已有大量 VMware 環境且授權可整合的情況下，vSAN / VxRail 是最省事的選擇；若需要傳統 SAN 效能，FlashSystem / PowerStore 配合 vSphere Plugin 也是常見組合。
💰
成本敏感 / 非核心應用
→ Ceph（開源）/ Longhorn
預算有限且有維運能力，開源 SDS 是最低成本選項；但需誠實評估人力成本，避免省了授權費但多花維運工時。
