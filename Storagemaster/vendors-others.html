Everpure（前 Pure Storage）— 全球市占第三#

Pure Storage 於 2026/02 正式更名為 Everpure，反映其從純儲存硬體廠商轉型為更廣泛的資料管理服務平台。產品名稱（FlashArray、FlashBlade、Evergreen）維持不變。

Evergreen 商業模式

Everpure 最大的差異化不在硬體規格，而在商業模式：

  • Evergreen//Forever：購買後控制器可原地升級，不需要搬移資料。截至 2026 年，97% 以上的銷售陣列仍在使用中
  • Evergreen//One（Storage-as-a-Service）：SLA 保證的按用量付費模式，有效容量不足自動補充
  • Evergreen//One for AI（2026 新推）：針對 AI 工作負載的彈性消費模式，解決 AI 基礎設施建置的不確定性

NetApp — ONTAP 生態最廣#

NetApp 的核心競爭力在於 ONTAP 作業系統橫跨所有硬體和雲端的統一生態，Block / File / Object / 雲端全部用同一套管理工具和技能組。

ONTAP 硬體產品線（2024～2025 全面更新）

系列定位代表機型協定特色
AFF A 系列 高效能統一儲存 A20 / A30 / A50 / A1K Block + File + Object 橫向擴展叢集，AI / VMware / 資料庫，即時 ML 勒索偵測
AFF C 系列 高容量 / 成本最佳化 C30 / C60 / C80 Block + File + Object 60 TB 磁碟，高密度；兩個機架可達 1.5 PB；QLC All-Flash
ASA r2（All SAN Array） 純 SAN 簡化平台 A20 / A30 / A50 / C30 FC / iSCSI / NVMe-oF（Block 專用） 對稱式 Active-Active 多路徑；SAN-only 簡化架構；ONTAP One 授權內含
AFX（新品） 高效能 NAS / S3 AFX NFS / SMB / S3 解耦架構（Disaggregated），NVMe-oF + RoCE 後端，AI / ML 工作負載
StorageGRID 大規模物件儲存 — S3 長期歸檔、備份目標、醫療影像；EB 級規模

ONTAP 生態的核心優勢

  • SnapMirror：最成熟的跨平台複製技術，支援同步（SnapMirror Synchronous）和非同步，可延伸至雲端（Cloud Volumes ONTAP）
  • SnapLock：WORM 不可變儲存，符合 SEC 17a-4 等金融法規合規要求
  • ONTAP Select（SDS 模式）：跑在通用 x86 上的軟體版 ONTAP，保有相同管理介面
  • Cloud Volumes ONTAP：AWS / Azure / GCP 上的 ONTAP，地端與雲端資料無縫移動
  • Trident（CSI Driver）：Kubernetes 原生儲存整合，是企業容器環境最常用的 NetApp 整合工具

HPE — GreenLake 消費模式#

HPE 的差異化策略是 GreenLake：所有產品都可以走按用量付費的消費模式，讓地端硬體像公雲一樣計費。儲存產品以 Alletra 系列為主力。

Alletra 系列

機型架構定位特色
Alletra MP B10000 解耦（DASE） 高效能 AI / 大規模 控制平面與資料平面分離，橫向擴展，NVMe-oF 後端，AI 工作負載導向
Alletra Storage MP 傳統雙控 中階企業 取代舊 Primera / 3PAR；GreenLake 原生；InfoSight AI 預測維運
Alletra dHCI 解耦 HCI 超融合延伸 計算和儲存可獨立擴展，不像傳統 HCI 必須同步擴展

InfoSight AI 預測維運

HPE 在 Nimble Storage 時代就開始的 AI 預測維運平台，現統一整合到 HPE GreenLake 雲端入口：

  • 跨客戶的遙測資料訓練 AI 模型，預測潛在問題
  • HPE 宣稱 86% 的問題在客戶察覺之前就已自動解決
  • 對台灣客戶而言，這個功能需要設備能連到 HPE Cloud，部分有資安要求的環境需要評估

GreenLake 消費模式

HPE 所有儲存產品都可走 GreenLake 按用量計費，依實際使用量計費，不需要預先購買固定容量。這對預算靈活度較高、或業務有明顯季節性波動的企業有吸引力。台灣企業對此模式的接受度正在提升中。

Peer Persistence（雙活）

HPE Alletra 的雙活功能，概念與 IBM HyperSwap 相近，需要 Quorum Witness 仲裁節點。Peer Persistence 與 VMware vSphere 的整合（vMotion、HA）被視為強項之一。

VAST Data — AI 時代新架構#

VAST Data 是目前企業儲存市場最受關注的新創廠商，採用完全不同於傳統設計的 Disaggregated Shared Everything（DASE）架構，主打 AI 工作負載。

DASE 架構的核心差異

傳統儲存陣列把計算（控制器）和儲存（磁碟）放在同一台設備。VAST Data 的 DASE 架構將兩者完全解耦：

  • Stateless 控制節點（CNode）：負責計算、協定處理、資料服務，無狀態設計，可以獨立橫向擴展
  • 儲存節點（DNode）：只負責儲存資料，可以獨立擴展容量
  • Global Namespace：所有節點共用同一個命名空間，Block / File / Object 統一，不需要分層設計

VAST Data Platform 特性

特性說明對標傳統方案
協定支援Block（NVMe-oF）/ File（NFS / SMB）/ Object（S3）/ Database（SQL API）四合一傳統方案通常需要不同設備
效能擴展計算和儲存獨立擴展，不互相綁定傳統雙控陣列計算/儲存同步擴展
AI 工作負載為 GPU 直接存取（GPUDirect Storage）最佳化，訓練 / 推理資料管線整合傳統 SAN 需要額外調優
資料縮減全域重刪 + 壓縮，跨整個叢集傳統方案通常在 Volume 或 Pool 層
台灣市場現況
VAST Data 在台灣目前主要出現在 AI 訓練基礎設施採購案（半導體、科技業），傳統企業（金融、製造）的滲透率仍低。但其架構概念代表了下一代企業儲存的設計方向，值得持續追蹤。
