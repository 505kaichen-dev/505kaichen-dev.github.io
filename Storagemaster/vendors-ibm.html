FlashSystem.ai — Agentic AI 管理

FlashSystem 新世代的核心差異化功能，AI 作為「共同管理員」自動執行日常儲存管理工作：

  • 自動化決策：每日執行數千個自動化決策（配置、調優、修復），減少人工管理約 90%
  • 勒索軟體偵測：FCM5 硬體層每 2 秒掃描 I/O 模式，偵測時間 < 1 分鐘，誤報率 < 1%
  • Storage Insights Pro 整合：主動監控與 AI 驅動的操作建議

FS5600 特有：SAS HDD 擴充支援

FlashSystem 5600 支援外接 SAS HDD 擴充機箱，7600 / 9600 不支援：

  • 4662-12G：12 槽 SAS HDD 擴充機箱，支援 12 / 20 / 24 TB SATA HDD
  • 4662-92G：92 槽高密度 SAS HDD 擴充機箱

此功能讓 FS5600 可建立 Flash + HDD 混合分層環境（SSD 作為熱層，HDD 作為冷層），適合成本敏感但仍需 All-Flash 效能的場景。

SAN Volume Controller（SVC）#

IBM SVC 是 IBM 的 In-band 儲存虛擬化平台，也是 FlashSystem 系列的管理軟體核心（FlashSystem 內建的軟體即為 Storage Virtualize，與 SVC 為同一軟體生態）。

SVC 與 FlashSystem 的關係
  • 獨立 SVC 設備（2145 系列）：作為外掛虛擬化層，管理外部異構儲存設備（Mdisk），提供統一的 VDisk 給主機
  • FlashSystem 內建 Storage Virtualize：FlashSystem 本身就跑 Storage Virtualize，可以同時作為底層儲存設備和虛擬化管理平台
  • 兩者軟體版本同步，功能集相同，差別在於硬體載體

DS8000（第 10 代）#

IBM DS8000 是針對 IBM Z 大型主機和 IBM i 系統設計的高端企業儲存，採用 FICON 和 FCP 介面，與 FlashSystem 是完全不同的產品線，不能混淆。

DS8000 vs FlashSystem 的定位差異
  • DS8000：專為 IBM Z / IBM i 設計，FICON 介面，與大型主機深度整合（ZHyperLink、GDPS），8 個 9 可用性，通常是金融業大型主機環境的標配
  • FlashSystem：開放系統（x86、AIX、Linux、Windows），FC / iSCSI / NVMe-oF 介面，是本知識庫的主要討論產品

第 10 代主要更新

  • FCM4 整合：首次將 FlashSystem 的 FlashCore Module 技術引入 DS8000，帶來硬體壓縮、加密與勒索偵測能力
  • PCIe Gen 4 I/O Bay：頻寬提升，高壓力交易場景延遲改善
  • ZHyperLink（PCIe Gen 4）：與 IBM Z 的超低延遲連接，比傳統 FICON 延遲降低約 20%
  • Safeguarded Copy：不可變備份功能，金融業合規要求
  • FIPS 140-3 認證：更高等級的加密合規

Storage Scale（前 GPFS / Spectrum Scale）#

IBM Storage Scale 是平行分散式檔案系統，前身為 GPFS（General Parallel File System），後改名 Spectrum Scale，現為 Storage Scale。適用於 HPC（高效能運算）和 AI 訓練資料集等需要大規模、高吞吐量的場景。

特性說明
架構橫向擴展分散式檔案系統，單一命名空間可達 PB 甚至 EB 級
協定支援NFS / SMB / S3 / POSIX，多協定同時存取同一檔案系統
效能特性高並行讀寫，適合大量小檔案（如 AI 訓練資料集）或大型循序 I/O（基因體、影像）
部署方式可部署在 IBM FlashSystem 後端、通用 x86 伺服器，或混合環境
主要競品Dell PowerScale（前 Isilon）、NetApp StorageGRID、VAST Data Platform

Storage Scale 不在本知識庫的主要討論範疇（重點為 FlashSystem / SVC Block 儲存），但在 AI 工作負載興起後，與 FlashSystem 搭配的場景越來越常見：FlashSystem 提供 Block 儲存（資料庫、VM），Storage Scale 提供高吞吐量的 AI 訓練資料儲存層。

Tape（TS4500 / TS7700）#

IBM Tape 產品在勒索軟體威脅升高的背景下，因其物理 Air-gap 特性重新受到重視，成為「最後一道防線」的資料保護方案。

產品類型主要用途最大容量
TS4500 磁帶庫 實體磁帶庫（LTO / 企業磁帶） 長期歸檔、Air-gap 備份、法規合規保存 支援至 LTO-10（40 TB / 卡匣），數千卡匣規模
TS7700 VTL 虛擬磁帶庫（大型主機） IBM Z 大型主機的磁帶虛擬化，磁碟 Cache + 實體磁帶後端 最大 1.6 PB 磁碟 Cache；附接 TS4500 時可達 100 PB

為什麼磁帶在 2026 年仍然重要

  • 物理 Air-gap：磁帶離線時無法透過網路存取，勒索軟體無法加密
  • 長期保存成本極低：磁帶的每 TB 儲存成本遠低於磁碟，適合 7～30 年的法規要求保存
  • 不可更改媒介：WORM（Write Once Read Many）磁帶提供不可變的物理保證
  • 能耗極低：磁帶離線時幾乎零功耗，適合綠色 IT 合規

Storage Ceph / Storage Fusion#

IBM 透過收購 Red Hat，將 Ceph 和 OpenShift 生態納入 IBM Storage 產品組合，提供雲原生和容器平台的儲存方案。

產品基礎支援協定主要場景
IBM Storage Ceph Red Hat Ceph Storage 商業版 Block（RBD）/ File（CephFS）/ Object（S3）三合一 OpenStack 私有雲、Kubernetes PV、大規模物件儲存、AI 資料湖
IBM Storage Fusion Red Hat OpenShift 原生儲存平台 Kubernetes CSI / PVC OpenShift 容器平台的儲存服務，整合 DR、備份、資料目錄

這兩個產品主要面向雲原生和容器化應用場景，與傳統 FlashSystem / SVC 的 Block SAN 場景不重疊。在台灣大型企業環境中，目前仍以 FlashSystem + SVC 為主，Ceph / Fusion 尚在導入初期。詳見 SDS 全景頁。
