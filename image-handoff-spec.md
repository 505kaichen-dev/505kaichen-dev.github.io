# 產圖需求規範 Image Handoff Spec

本文件供 Codex 撰寫 HTML 時自動補入圖片標記，以及產出移交給 ChatGPT 的 TXT 檔案使用。GPT 收到 TXT 後依規範產圖。

---

## 一、圖片標記類型

HTML 中需要圖片素材的位置，一律使用 HTML 註解插入標記。共四種類型：

| 類型 | 用途 | 尺寸規則 |
|------|------|----------|
| `[COVER]` | 頁面封面、OG 圖、社群分享圖 | 固定 **1200x630**，不得更改 |
| `[IMAGE]` | 文章配圖、教學示意圖、情境插畫 | 預設 800x600，可依版面指定 |
| `[CHART]` | 流程圖、架構圖、比較圖、資訊圖表 | 可不標尺寸 |
| `[ICON]` | 功能圖示、品牌圖示、客製插畫圖示 | 可不標尺寸 |

**禁用類型：** `HERO` / `BANNER` / `THUMBNAIL` / `ARTICLE-COVER` 一律不得使用。  
**禁用舊格式：** `<!-- [插圖] 尺寸：＿＿px 內容：＿＿ -->` 一律不得使用。

---

## 二、標記格式

```html
<!-- [COVER] 1200x630 圖片描述 -->
<!-- [IMAGE] 800x600 圖片描述 -->
<!-- [IMAGE] 600x400 圖片描述 -->
<!-- [CHART] 圖片描述 -->
<!-- [ICON] 圖片描述 -->
```

---

## 三、圖片描述撰寫規則

每條描述必須包含以下五個要素，缺一不可：

1. **畫面主體** — 畫面中心是什麼
2. **情境** — 發生什麼事、什麼場景
3. **視覺風格** — 插畫、扁平、手繪、寫實、科技感…
4. **具體色票** — 必須寫出十六進位色碼，例如 `#6c47ff`、`#F7F4EE`
5. **人物設定**（如有）— 性別、年齡感、動作

**描述語言統一使用繁體中文。**

### 禁止寫法

```html
<!-- 太模糊，沒有風格和色票 -->
<!-- [IMAGE] AI工具介紹圖 -->

<!-- 只說跟隨 CSS，GPT 無法理解 -->
<!-- [IMAGE] 風格跟隨本頁 CSS 設計語言 -->
```

### 正確寫法

```html
<!-- [IMAGE] 800x600 開發者在多螢幕工作環境中整理 AI 工具流程，
     藍白科技感介面（#2563EB / #F8F9FC），玻璃擬態風格，
     年輕男性側臉，專注看螢幕 -->
```

## 四、標記放置位置

### COVER
- 每個子頁面**至少一個**，放在 `<body>` 開頭、HERO 區塊之前
- 依該頁面主題設計，不得套用首頁卡片文案

### IMAGE / CHART / ICON
- 放在**該段落內容結尾、下一個區塊開始之前**
- 視覺密度已高的區塊（互動元件、表格、程式碼框密集處）可省略
- 每個主要章節最多一張，避免過度堆疊

## 五、Codex 完成 HTML 後的動作

### 步驟 1：在 HTML 末尾加入產圖移交摘要

```html
<!--
=== 產圖移交摘要 ===
本次頁面圖片需求如下，供 ChatGPT 掃描使用：

[COVER] 1200x630 ...
[IMAGE] 800x600 ...
[CHART] ...

==================
-->
</body>
</html>
```

### 步驟 2：產出獨立 TXT 檔案

- 檔名格式：`{資料夾名稱}_圖片移交.txt`
- 例：`ai-guide_圖片移交.txt`、`opensource-story_圖片移交.txt`
- 內容只包含摘要區塊，**不含任何 HTML 標籤**
- 此 TXT 直接移交給 ChatGPT，不需傳整份 HTML

#### TXT 內容格式範例

```
=== 產圖移交摘要 ===
本次頁面圖片需求如下，供 ChatGPT 掃描使用：

[COVER] 1200x630 「代碼烏托邦」開源精神故事封面圖，雜誌長文風格，
暖米色背景（#F7F4EE），中央大標題「他們把改變世界的程式碼免費送給了所有人」，
左下角散落 Linux 企鵝、Git 分支圖示、GNU 標誌等開源符號，
藍色強調色（#2A5C8F），整體質感沉穩典雅如科技人文雜誌封面

[IMAGE] 800x500 開源知識共享概念插圖，一顆發光的程式碼球體從一雙手中
向外輻射擴散至全球無數人，線條扁平插畫風格，暖米色背景（#F7F4EE），
藍色（#2A5C8F）光芒與連線，象徵知識自由流動

[CHART] Memcached DDoS 攻擊原理示意圖，「攻擊者」送出小型請求至
「Memcached 伺服器群」，放大 51200 倍後形成洪流打向「GitHub」，
旁邊標示「1.35 Tbps」峰值，暖黃背景（#FDF8F0），
紅橙色（#B85C38）危險節點，藍色（#2A5C8F）正常節點，扁平箭頭流程圖風格

==================
```

## 七、給 ChatGPT 的產圖說明

收到 TXT 後，請依以下規則產圖：

1. 每個標記產出一張圖
2. `[COVER]` 固定輸出 **1200×630px**
3. `[IMAGE]` 依標記中的尺寸輸出，未標示則預設 **800×600px**
4. `[CHART]` 以資訊圖表方式呈現，強調清晰易讀
5. `[ICON]` 輸出方形透明底 PNG，尺寸 512×512px
6. 色票以標記中的十六進位色碼為準，不得自行替換
7. 描述中的中文文字內容（標題、標籤）保留繁體中文
8. 人物不得出現可辨識的真實人物面孔
