# 發布與同步檢查表

## 修改前

- [ ] 確認目前 Git 工作樹已有的修改與刪除，不覆蓋其他人的工作。
- [ ] 確認內網版是本次修改來源。
- [ ] 若要搬檔，先搜尋完整檔名與相對路徑引用。

## 修改後

- [ ] 所有正式專案仍有 `index.html`。
- [ ] 根首頁卡片的資料夾、封面與分類正確。
- [ ] 本機 `href`、`src` 沒有失效。
- [ ] 手機與平板至少檢查一種直向及橫向尺寸。
- [ ] 新圖片符合 `image-handoff-spec.md`。
- [ ] 新專案有 `PROJECT.md`。
- [ ] 更新 `website-maintenance.md` 卡片序號表。
- [ ] 更新 `docs/PROJECTS.md` 與必要的交接文件。
- [ ] 封存動作記錄於 `docs/ARCHIVE_LOG.md`。

## GitHub 同步前

- [ ] 不提交 `_archive/` 內不必要的大型檔案，除非確定需要版本保存。
- [ ] 不把 TTS、提示詞工作區等內部工具放回公開網站根目錄。
- [ ] 不提交 `Thumbs.db` 或作業系統暫存檔。
- [ ] 檢查大於 2 MB 的新圖片是否需要壓縮。
- [ ] 軟體下載使用外部雲端分享，Pages 工作區不提交大型 EXE。
- [ ] 軟體新版本同步更新外部連結、版本、大小與 SHA-256。
- [ ] 使用瀏覽器從首頁逐一開啟新增或修改的入口。
